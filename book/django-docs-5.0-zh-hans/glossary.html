术语¶

具体的模型¶
一个非抽象 (abstract=False) 模型。
字段¶

一个属性在 model 中; 一个给定的字段通常被映射到数据库表的一列上。

参见 模型。

通用视图¶

高阶的 view 函数针对开发中常见的习惯用法或模式，提供了一种抽象/通用的实现。

参见 :doc: /topics/class-based-views/index。

模型¶

模型存储你的应用程序的数据。

参见 模型。

MTV¶

"模型-模板-数据"("Model-template-view"); 一种软件模式，风格类似于MVC，但是更好地描述了 Django 所做的事。

参见 the FAQ entry。

MVC¶
模型-视图-控制器; 一种软件开发模式。Django 基于 MVC 做了某些扩展。
项目¶
一个 Python 包 —— 即一个代码目录 -- 它包含 Django 的一个实例中所有的设置。这包括数据库配置，Django 的特定选项和特定应用程序设置。
属性¶

也被称为 "管理属性"，自 Python 2.2 版本起支持的功能。这是用一个巧妙的方法来实现的属性，其用法类似于属性的访问，但实现是调用方法。

参见 property。

查询结果集¶

一个对象表示了从数据库中取到的某些行。

参见 执行查询。

斜杠¶

作为某些东西的简短标签，只能包含字母，数字，下划线或连字符。他们一般被用于 URL。例如，在一个典型的博客文章的网址：

https://www.djangoproject.com/weblog/2008/apr/12/spring/

最末尾 (spring) 是一个斜杠。

模板¶

一个文本块，作为用来表示数据的格式。模板有助于从数据本身抽象地展示数据。

参见 模板。

视图¶
负责渲染页面的函数。
